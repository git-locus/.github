# Contributing to Locus

## Workflow

1. Apri una issue prima di iniziare lavori significativi.
2. Crea un branch dal `main` con naming `feat/`, `fix/`, `chore/`.
3. Apri una Pull Request collegata alla issue.
4. Almeno un review è richiesto prima del merge.

## Commit messages

Usa il formato [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: aggiunge autenticazione OAuth
fix: corregge errore 500 su /api/posts
chore: aggiorna dipendenze npm
```

## Local development

Vedi [`docker-compose.yml`](docker-compose.yml) e la cartella [`knowledge/`](https://github.com/git-locus/knowledge) per istruzioni su come avviare l'ambiente locale.
